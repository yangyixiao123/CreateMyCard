# 布局系统

当字号预算失败、组件重叠、内容碰撞、连续空洞超过阈值，或紧凑布局宽高无法推导时读取本文档。本文只用于内部规划，不要输出布局计划对象。

## 单文件兜底

- `manifest`：指选中模板目录内的 `manifest.json`；未使用模板时忽略 manifest 槽位。
- `mustKeep`：没有它卡片无法回答需求；必须完整显示。
- `shouldKeep`：空间允许才保留；不得挤占 `mustKeep`、受保护文本或动作热区。
- `metric/tile/status/badge`：模板声明的并列小事实槽位，不按普通支撑条数折算。

## 先算这些（必须先过）

- root：先确定逻辑画布、安全区和内容区，`2x2` 按 `160vp x 160vp` 预算、内容区约 `136vp x 136vp`，`2x4` 按 `320vp x 160vp` 预算、内容区约 `296vp x 136vp`。`createSurface` 和 root `width/height` 必须写 `"matchParent"`。
- 横向：每个 Row 的固定宽度、文本估算宽度、子项左右 `margin`、padding 和 `itemMargin` 相加后，不超过父容器宽度。
- 纵向：每个 Column 的元素高度、文本行高预算、子项上下 `margin`、padding 和 `itemMargin` 相加后，不超过父容器高度。
- 文本：每个受保护文本都有字体大小、行数、宽度和完整显示策略。
- 动作：每个 Button 有不小于 `24vp` 的点击视觉尺寸；和文本并排时，父 Row、Text、Button 都要有明确宽高预算。
- 层叠：每个 Stack 都能说明背景层、信息层和点击层，且不会覆盖受保护文本、CTA 或主数值。
- 对齐：同一功能组的背板、文本、按钮共享边界；堆叠元素之间仍保留 4-12vp 间距。
- 底部：支撑/动作区外框锚定安全区底边，没有漂浮；按钮和相邻文本视觉基线一致。

如果任何一项算不清，先删减 `shouldKeep` 信息或换更简单布局，再考虑升级尺寸；不要用省略、裁剪、默认伸缩或 Stack 覆盖来补救。

## 画布、尺寸和区域

- 基础尺寸、root 圆角、clip 和安全区以 `core-rules.md` 为准；本文件只补布局判断。
- root `width` / `height` 默认写 `"matchParent"`；内部主区域、背板、按钮和进度图形若承担固定格式，仍要写稳定数值 `width` / `height` / `constraintSize`，不要让动态文本自由撑开画布。
- 端侧真实父布局可能存在设备差异；云侧 DSL 不按设备枚举改内部数值，端侧按基准逻辑画布做整体映射。
- `2x2` 非模板默认最多 3 个主区域和 1 个显式动作；默认结构是 1 个主视觉/主数值区、1 个支撑行、可选 1 个动作。使用正式模板时按 manifest 的 `sections` / `slots` 上限执行，不新增第二服务对象或第二动作目标。
- `2x4` 最多 4 个主区域和 2 个动作区；常用横向分区是 `180-192vp` 主内容 + `72-84vp` 动作/状态 + `8-12vp` 间距，或两个 `139-145vp` 左右的对等区。
- 未指定尺寸先尝试 `2x2`；升级 `2x4` 必须有具体失败点：受保护文本/热区放不下、核心内容需要并列理解、关键媒体必须同屏，或用户明确要求横版/并排/双面板。
- 用户指定 `2x2` 时先删 `shouldKeep` 信息、减少区域、收敛动作；指定 `2x4` 时保持一个主对象，增加真实支撑信息、可读宽度或动作标签/热区，不简单放大 `2x2`。
- 请求 `1x2`、`4x4` 或其它未声明尺寸时，不生成该尺寸；改给最接近的 `2x2` 或 `2x4` 替代方案。
- 使用固定比例带，不要临场发明比例：`62/38`、`55/45`、`50/50`、`30/70`、`15/85`。数字/状态常用 `55/45`，时间线常用 `30/70`，媒体常用 `62/38`。
- 信息文本、图标、按钮、数据图形、highlight 和背板必须在安全区内。背景图片或渐变允许贴边，但不能承载必须读取的信息。
- 固定间距尺度：只使用 `2/4/6/8/10/12/14/16`；优先 `4/8/12/16`，`2` 只做微调，`10/14` 只做严密预算修正。
- 间距默认：`2x2` 组内 4-8vp、组间 8-12vp；`2x4` 组内 6-8vp、组间 8-12vp；背板内 padding 8-10vp。不要套多层内部卡片。
- 先定义标题/状态组、主指标组、支撑信息组、动作组；同组内部间距小，跨组间距大。
- 内部半透明块用 14-18 圆角，小图标块用 5-8，pill/action bar 取 height/2；浅色卡只用低 alpha 黑色阴影，例如 `#22000000`。
- 默认左对齐。只有内容不超过 1 个主显示组 + 1 个短支撑组，且没有显式动作区时，才使用居中。

## 预算顺序

写 JSON 前按这个顺序过一遍：

1. 定 root 外围尺寸、安全区和内容区；内容预算按 `2x2 = 160vp x 160vp` 或 `2x4 = 320vp x 160vp`。
2. 定主区域、支撑区域、动作区域的固定宽高；`2x2` 默认只允许三段，`2x4` 默认只允许左右两列或三段横向组。
3. 先给图标、进度、按钮等固定元素占位，再把剩余宽度分给文本。
4. 按文字估算规则确认受保护文本能放下；放不下先删可选信息，再降字体或换尺寸。
5. 确认所有 gap、padding、margin、itemMargin 后，横向总和不超过父宽，纵向总和不超过父高。

常用安全槽位：

- `2x2 top-hero-bottom`：顶部 18-24vp，间距 4-6vp，hero 50-62vp，间距 6-8vp，底部 26-34vp，总高不超过 136vp。
- `2x2 icon-fact-action`：图标/环 56-72vp，事实区 28-38vp，动作 28-34vp；三者不能都取上限。
- `2x4 split-action`：主内容 180-192vp，动作/状态 72-84vp，中间 8-12vp；左右列都要写稳定宽度。
- `2x4 media-info`：媒体/主视觉 96-116vp，信息列 154-172vp；需要动作区时将信息列收敛到 104-128vp、动作区 56-72vp。媒体不能挤压标题和 CTA。

## 字体阶梯

- 只使用批准阶梯：`10fp`、`12fp`、`14fp`、`16fp`、`18fp`、`20fp`、`32fp`、`40fp`。
- 字体层级先按职责定角色，再选阶梯；同一卡片控制在 3 档以内：主视觉/主数值、标题/状态、支撑信息。
- hint / 占位式提示：`10fp`，只用于非主信息、刻度、弱提示；不要用作主信息。
- 支撑信息和 metadata：`12fp`，字重 Regular/Medium。
- 紧凑标题、普通正文、短状态：`14fp`，字重 Regular/Medium；重要标题可用 Semibold。
- 主标题/正文：`16fp` 或 `18fp`，只在空间足够时使用。
- 紧凑数值：`20fp`；重要数值：`32fp`；单一绝对主数值：`40fp`。
- CTA：`14fp Medium`，文字保持一行，优先 2-6 个中文字符。
- 同一卡片只保留一个最强字重；不要把所有标题、数值、按钮都设成 Bold。
- 如果层级不清，先降低支撑文本字重或删 `shouldKeep` 信息，不要继续放大多个元素。
- 预算垂直高度时按 `fontSize + 2-4fp` 计算。不要为了行高去输出组件目录未列出的属性。
- 系统大字体适配时，`20fp` 以上主数值不继续放大；普通文本放大上限约 `1.15x-1.3x`。

## 文本适配

- 受保护文本必须完整显示：标题、日期、星期、时间、CTA、状态、主指标、倒计时、价格/数量和入选用户字段。
- 写 JSON 前先估算宽度：`available = parentWidth - horizontalPadding - gaps - fixedIconOrDividerWidths`。
- 估算文本宽度时，中文字符约等于 `fontSize`，英文/数字约等于 `0.6 * fontSize`，空格和标点约等于 `0.3-0.5 * fontSize`。
- 受保护文本放不下时，按顺序处理：缩短周边标签，删掉可选字段，换成两行且预留高度，降低到同阶梯下限，或选择 `2x4`。
- 不要依赖 `ellipsis`、`clip` 或 marquee 隐藏受保护文本。可选文本与其截断，不如删除或移动到详情。
- 日期/时间、状态、CTA 不要同时挤在一个窄 Row 内，除非每个元素都有明确宽度预算。

## 重叠防线

- 优先使用 Row/Column 正常流；只有背景层、进度环居中、角标或明确叠加时才使用 Stack。
- Stack 中背景放底层，内容放上层；信息层必须有稳定宽高、对齐方式和 padding，不能靠视觉猜测避让。
- Stack 叠加不得覆盖受保护文本、CTA、点击视觉元素或进度主值。
- 固定高度/宽度容器内的弹性子项必须可收缩或被约束；在 DSL 中用明确 `width`、`height`、`constraintSize`、`flexShrink` 或容器 `clip` 兜底，避免内容压力把元素推出安全区。
- 使用 `spaceBetween` 时，左右两侧都要有可承受的宽度；动态长文本不要和固定按钮抢同一行。
- 重复项必须有稳定高度和间距；`List` 只用于少量真实重复项，避免滚动和密集列表。
- 每个按钮或可点击视觉元素不小于 `24vp`；周围布局目标热区为 `40vp`，空间不足时必须保留不小于 `24vp` 的点击视觉尺寸。
- 同一功能组内，内容背板、文本块、按钮要共享左右边界。堆叠按钮直接服务某个内容背板时，默认和背板等宽，不要窄按钮挂在宽背板下方。

## 图标布局

- 使用素材库图标时，必须显式设置 `width`、`height` 和 `objectFit: "contain"`；不要让图片自由撑开 Row/Column。
- 辅助语义图标默认 16-24vp；主视觉图标在 `2x2` 默认 44-64vp，在 `2x4` 默认 48-72vp。
- 当图标本身是卡片识别身份（天气、信号、电池形状等有约定俗成图形语言的状态）时，给它独立主视觉区，尺寸不小于附近主数值字号的 `1.5x`，或不小于 `44vp`；不要把它作为 16-20vp 的行内小图标。
- 自绘状态图标的内部形状用图标盒子的百分比定位；不同尺寸只改外层 `width` / `height`，不要用 `transform: scale()` 缩小，因为缩放不会减少布局占位。
- 图标 + 文字同 Row 时，先扣除图标宽度和 4-8vp gap，再估算文本宽度；不要把长标题和固定图标硬塞同一行。
- 图标只服务识别、状态、动作或主媒体角色；没有绑定这些角色的图标不使用。若加入图标会导致 CTA、时间、标题或主指标截断，放弃图标。
- 同组多个小图标必须尺寸一致、基线一致；不要在紧凑卡片中混用 16vp、24vp、40vp 多级小图标。

## 进度与图表几何

是否使用 ring、bar 或普通文本由 `generation-workflow.md` 先判定；本文只管几何和预算。

- ring 用于单一目标/范围且是主焦点。`2x2` 最多一个主 ring；`2x4` 可用一个主 ring 搭配支撑文本，不要左右各一个 hero ring。
- ring 必须是正圆，显式设置相同 `width` / `height`。`2x2` 常用 56-72vp，`2x4` 主 ring 常用 64-88vp；stroke 约为 ring 直径的 14-15%。
- 数值默认放在 ring 旁边或下方作为主数值文本；不要把多字符主数值塞进小环洞。环洞内只放简短图形或功能 glyph。
- repeated small ring 只用于 2-4 个同类比例项；每项小环约 32-40vp，标签承载名称和值。超过约 4 项或需要精确比较时改用 linear bar。
- step / segmented bar 用于有序阶段。每段等宽，段间 2-4vp，当前阶段要配文字标签，不能只靠段数表达状态。
- linear bar 用于多个可比较比例。track 与 fill 高度统一为 6-8vp，fill 必须有明确宽度，不要让布局自动压成 0。
- 图表颜色只使用场景主色、状态色或中性 track；颜色来源和前景配对按 `color-token-system.md`。

## 布局原型

- `hero-top`：适合 `2x2`。上方标题/状态 18-24vp，中间主视觉/主数值 32-44vp，下方支撑或动作 22-32vp。
- `hero-left`：适合 `2x4`。左侧主视觉/指标 72-96vp，右侧放 2-3 条事实；不要两侧都做主视觉。
- `split-action`：适合 `2x4`。主内容 180-192vp，动作/状态 72-84vp，中间 8-12vp 间距。
- `single-panel`：中性 root + 一个内容背板，适合日程、备忘、票券、账单、关怀、物流；背板 padding 8-10vp，不要再嵌套卡片。
- `meter-focus`：一个进度/仪表视觉 + 紧凑标签。进度环在 `2x2` 用 56-72vp，在 `2x4` 用 64-88vp；数值不要塞进很小的环洞。
- `ambient-root`：root 承载天气、睡眠、音乐或环境场景色；主文本必须落在中性背板、遮罩层或通过 `color-token-system.md` 校验的 on-color 前景上。

## 2x2 容量

`2x2` 容量按区域和角色槽位判断，不按普通文字条数判断。非模板默认是一个标题/状态区、一个主视觉/主数值区、一个支撑/并列事实区和可选一个动作；正式模板按 manifest 承载两个 metric、两个 tile、2-3 个 status ring、一个 badge 或图标动作。

常用区域关系只保留四类：

- 主值 + 支撑/动作：顶部对象，中间 `32/40fp` 主值或主图标，底部支撑背板或动作胶囊。
- 仪表 + 事实：一个主 ring/bar，旁边或下方放短事实，底部可保留动作。
- 并列小事实：两个 metric/tile 或 2-3 个小状态环必须同尺寸、同间距，不抢主显示组。
- 媒体/氛围：图片、渐变或场景色只作 root/媒体层；文字落在中性背板、遮罩层或合法 on-color 前景上。

若必须同时保留两个服务对象、两个主任务、两个动作目标，或长标题 + 媒体 + 详情，才考虑 `2x4`。

## 背板和动作关系

- `2x4` 右侧支撑组如果是多行散点信息、小矩阵、多个 label/value，要放入一个中性背板；不要让它们散落在 root 上。
- 单个孤立元素不需要背板：一个按钮、一个状态行、一个图标如果没有同组内容，单独包背板会增加噪声。
- 有支撑文本 + 一个动作的面板，按钮贴近面板底部并与内容同宽；按钮上方必须有真实内容填充。
- 如果按钮上方只有一行短文本和大片空白，放弃底部锚定：加真实支撑行、拓宽内容列，或把内容和按钮作为一个居中组。
- 底部支撑/动作区服务整张卡片时，它的外框应贴近安全区底边。`2x2` 外框底边到卡片底边保持 `8-12vp`，最大不得超过 `16vp`；`2x4` 保持 `10-14vp`，最大不得超过 `16vp`。不要只依赖 `spaceBetween` 的几何分布；若按钮下方留白大于按钮上方相邻间距，或动作区中心线距离安全区底边超过 `24vp`，则下移动作区、增高动作区或重分配上方内容高度。
- 底部操作区必须与上方主内容、进度条或支撑背板共享同一左右边界；不要让按钮组比它服务的内容明显窄、偏移或单独悬挂。
- 紧凑 Row 中按钮与小号文本并排时，Row 高度必须同时容纳文本估算行高和按钮高度；若 `Button.height - textLineHeight > 8vp`，优先调整按钮槽位高度或上下 padding。只有父 Row 高度仍能容纳 `Button.height + margin.top/bottom` 时，才用 `margin.top` / `margin.bottom` 做 `1-2vp` 校正。

## 留白处理

- 默认连续留白控制在 6-12vp；超过 18vp 的连续空洞必须绑定媒体、进度图形、场景背景或可点击热区之一。
- 如果主视觉/主数值区低于内容区面积约 40%，且只剩一行小字，放大主显示组、加入 1 条已入选 `shouldKeep` 支撑信息，或改用 `2x2`。
- sparse 内容的主对象高度目标为可用高度的 `75-85%`，但仍必须使用批准字号、图标和间距阶梯。
- `2x4` 若只有 1 个短标题且没有媒体、进度、支撑组或动作区，应改用 `2x2`；若必须保持 `2x4`，使用左右分区、背板组或动作区承载真实内容。
- 不要增加没有绑定 `mustKeep`、`shouldKeep`、状态、动作或主媒体的装饰。优先调整比例、删减区域、放大主显示组。

## 输出前检查

- 间距：每个 gap、padding、itemMargin、margin 都能回溯到固定尺度；组间距大于组内距。
- 密度：没有滚动、长列表、表格、按钮网格、密集多面板，且不存在未绑定媒体/进度/场景背景/热区的 `>18vp` 连续空洞。
