# 能力范围

## 支持的卡片任务

此 skill 用于根据一句话生成紧凑 HarmonyOS GenUI 桌面卡片，尤其适合：

- 天气 + 日程速览：当前状况、下个会议、快捷动作。
- 家庭关怀/天气告警：温度、风险标签、呼叫或关怀动作。
- 会议/日程提醒：下个事件、时间段、房间/地点、专注动作。
- 睡眠/健康进度：状态、环形进度、时长、提醒动作。
- 倒计时/训练计划：事件倒计时、今日计划、打开计划动作。
- 设备/产品状态：产品图片/资源、电量、模式、播放、连接状态。
- 通用状态卡：一个主指标、一个辅助上下文、一个动作。
- 紧凑信息摘要：把一个实体/概念/地点/事件总结成卡片，而不是文章。

默认输出是使用 `ohos.a2ui.extended.catalog` 的单个桌面卡片 surface。

## 支持的尺寸

### `2x2` / `160 x 160vp`

用于回答一个即时问题的速览卡片：

- 当前状态是什么？
- 接下来是什么？
- 现在应该做什么？
- 进度到哪里了？

### `2x4` / `320 x 160vp`

用于需要更大横向空间但仍保持卡片形态的紧凑横版卡片：

- 双面板日程或提醒。
- 状态加联系动作的关怀/建议卡。
- 媒体加状态的产品/设备卡。
- 主视觉加 2 到 3 个辅助详情的进度卡。

## 不支持或有风险的请求

当用户要求以下内容时，说明限制并提供一个更小的受支持卡片替代方案：

- 完整页面、多屏流程、重型仪表盘或文章/详情页。
- 需要大量输入、校验或大型动态列表的复杂表单。
- GenUI standard 或 HarmonyOS extended catalog 之外的组件。
- 需要不受支持的绝对定位或 CSS 类属性的布局。
- 实时数据获取、设备 API，或应用未提供的宿主函数。
- 用户没有提供、且不能用本地资源表示的远程图片或媒体。
- 无法在 `2x2` 或 `2x4` 中压缩而不损失用户核心需求的内容。

## 默认假设

- 卡片尺寸：默认选择 `2x2`，除非请求明显需要 `2x4`。
- 宽度预算：`2x2` 为 `160vp`，横版 `2x4` 为 `320vp`。
- 高度预算：`2x2` 和 `2x4` 都是 `160vp`。
- 协议：包含 `createSurface`、`updateComponents`、`updateDataModel` 的 JSONL。
- Catalog：`ohos.a2ui.extended.catalog`。
- 内容密度：`2x2` 有 2 到 3 个主区域；`2x4` 有 3 到 4 个主区域。
- 交互：默认最多一个主动作，除非用户明确需要更多。

## 替代策略

如果请求过宽，生成一个回答首要速览需求的聚焦卡片：

- 旅行计划卡 -> 下次旅行倒计时 + 天气 + 打开行程。
- 财务仪表盘卡 -> 今日支出/预算状态 + 风险标签 + 查看详情动作。
- 项目管理卡 -> 下个里程碑 + 进度 + 打开任务动作。
- 文章摘要卡 -> 标题 + 2 个属性 + 一行摘要 + 打开详情动作。
