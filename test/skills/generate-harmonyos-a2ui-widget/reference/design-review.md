# 设计评审

在校验器通过后使用本文档。所有修复都必须使用 HarmonyOS GenUI extended 样式和组件。

## 紧凑卡片检查

- 卡片回答一个一眼可读的问题。
- 选择的尺寸明确：`2x2`（`160 x 160vp`）或横版 `2x4`（`320 x 160vp`）。
- `2x2` 主视觉区域不超过 3 个。
- `2x4` 主视觉区域不超过 4 个。
- 主指标/标题明显占主导。
- 次要上下文短于主内容。
- 动作明显，但不大于主信息。
- 没有长正文、文章章节、表格和重型列表。
- 卡片从可泛化规则构造，而不是从模板/示例产物复制。

## 布局检查

- Root 有卡片式 shell：圆角、裁剪，以及清晰背景或图片。
- Root 尺寸适配所选目标。如果 width 是 `"100%"`，内部布局仍然遵守 `2x2` 的 `160vp` 或横版 `2x4` 的 `320vp`。
- 横向行有 2 或 3 个直接子节点，除非使用有明确理由的重复项模式。
- 时间、CTA、状态标签、价格或百分比等受保护文本没有使用狭窄固定宽度。
- 弹性文本列使用 `layoutWeight`；小图标和固定圆形使用明确宽高。
- `maxLines` 和 `minFontSize` 用于保护受约束文本，同时不隐藏关键信息。
- 关键信息不能靠截断“保护”。日期、星期、时间、状态、CTA、主指标、主标题和用户要求字段必须完整显示，而不是用 ellipsis、clip 或 marquee。
- 拥挤 Row 有明确宽度预算；分隔线、间距、padding 和固定列没有占用受保护文本所需空间。

## 视觉检查

- 色板贴合当前场景，并且是为当前场景选择的。
- 渐变和光晕层支持内容，而不是纯装饰。
- 渐变或图片上的文字对比足够强。
- 半透明块有边框或背景对比。
- 如果使用图片，它是真实资源，且重要主体没有被文字遮住。
- `2x2` 只有一个主导视觉锚点。
- `2x4` 有一个主导视觉锚点，最多一个次级视觉装置。

## 交互检查

- 可点击视觉区域有 `onClick` 或 `Button.action`。
- 宿主函数名有意义且稳定。
- 必要动作参数从 DataModel 绑定；业务 ID 不硬编码。
- 主指标、动作和有意义的纯图标区域有 accessibility label。

## 数据检查

- DataModel 键是语义化的，并按领域分组：`weather`、`meeting`、`action`、`asset` 等。
- 当本地化或格式化会让表达式脆弱时，预先计算展示字符串。
- 没有假 URL，也没有从示例遗留的业务样例事实。

## 最终检查

任何编辑后都重新运行校验器：

```bash
python scripts/validate_genui_card.py path/to/card.dsl.jsonl
```
