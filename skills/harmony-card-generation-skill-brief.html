HarmonyOS A2UI Form · Create My Widget 方向

一句话生成鸿蒙桌面卡片：Skill 设计思想精简说明

这个 skill 的目标不是让大模型自由画 UI，而是把自然语言需求编译成端侧可解析、可渲染、可刷新、可评审的卡片结果： 一个 genui DSL JSONL 加一个 cardspec 数据能力契约。

1. 核心定位

生成对象

一张 Harmony Form 服务卡片

只生成 2x2 或横版 2x4 卡片，不扩展成页面、文章、表格或复杂表单。

双产物

DSL + 数据契约

genui 负责组件渲染和交互；cardspec 负责端侧数据能力、写入路径和持久化刷新依据。

方法论

规则构图，不套模板

从用户语义、尺寸预算、组件白名单和数据能力 manifest 推导卡片，而不是复制历史样例。

2. 目标链路

用户输入小艺 App 接收自然语言需求，如通勤、天气、日程、关怀。
云侧 Agent按 skill 判断场景、尺寸、能力边界和语义角色。
输出结果生成 genui JSONL 与 cardspec JSON。
端侧渲染A2UI 解析 DSL，创建 surface，渲染组件树。
运行时刷新端侧执行 CardSpec 数据能力，并通过 updateDataModel 刷新。

3. 为什么要分成 genui 和 cardspec

产物 职责 端侧价值
genui A2UI Form JSONL：createSurface、updateComponents、updateDataModel。 端侧可直接解析渲染，并用 DataModel 驱动 UI 响应式更新。
cardspec 卡片尺寸和数据绑定契约：suggestSize、dataBindings、capabilityId、arguments、writeResultTo。 端侧知道该查什么、参数是什么、结果写到哪里，后续可持久化和定时刷新。
关键点： CardSpec 不是附件，而是同一张卡片的运行时契约。点击事件不写入 CardSpec，只进入 DSL 的 onClick。

4. 协议收敛策略

允许什么

使用 version:"v0.9"、ohos.a2ui.extended.catalog、扁平组件邻接表、一次完整 updateComponents， 以及 10 个 Form 组件：Text、Image、Divider、Progress、Button、Checkbox、Row、Column、List、Stack。

禁止什么

禁止未声明组件、标准 catalog 属性名、theme、Button.action、网络图片、SVG、表达式 {{...}}、$__dataModel、$__widthBreakpoint 等。

Text.content Image.src Button.label onClick only {path:"/..."} formatString updateDataModel 预计算展示字段

5. 卡片构图规则

尺寸 适用场景 构图约束
2x2
160 x 160vp		 一个主答案加一个辅助上下文，如天气、下个会议、低电、倒计时、简单动作。 主区域不超过 3 个；默认不滚动；root padding 约 10 到 12vp；保护时间、温度、CTA、主标题等关键文本。
2x4
320 x 160vp		 需要左右关系或更安全横向空间，如日程加动作、设备媒体加状态、天气加待办。 主区域不超过 4 个；通常是 focal pane、detail pane、action/status；避免长列表和页面化布局。
保护文本原则： 日期、星期、时间、倒计时、温度、百分比、状态、CTA、主标题和用户明确要求字段必须完整显示，不能用省略号、裁剪或逐字换行掩盖布局失败。

6. 数据能力设计

能力 manifest

当前已有天气和日历

weather.overview.get 用于当前天气、预报、空气质量等；calendar.events.search 用于本地日历事件查询。

DataModel 映射

writeResultTo 决定 UI 路径

例如日历写到 /data/calendar，UI 列表绑定 /data/calendar/items；天气写到 /data/weather。

隐私边界

初始值只放空结构

DSL 初始化 loading、空对象和空数组，不写用户真实日程、位置或天气。真实数据由端侧执行能力后注入。

扩展方式： 新增 Todo、通勤、设备状态、健康、应用使用时长等能力时，继续沉淀为 reference/data-capability/ 下的 manifest，Agent 按声明选择能力，不在提示词中硬编码。

7. 点击事件设计

点击能力由 reference/event-capability/click-event.md 约束。当前支持拨号、打开声明过的应用页面、查看日程详情。

functionCall 用途 核心约束
clickToCallPhone 跳转拨号界面 只允许参数 phonenumber。
clickToDeeplink 打开设置、天气、闹钟等已声明页面 bundle、ability、uri 必须来自 supportedTargets。
clickToIntent 查看日程详情 intentName 固定为 ViewCalendarEvent，entityId 从日程数据绑定。

8. 质量闭环

生成前

先形成布局理由

明确尺寸选择、语义角色、视觉焦点、素材来源、宽度预算、交互来源和 CardSpec 映射。

生成中

至少一次改进

对首版草稿做层级、紧凑度、场景特征或关键文本完整显示的显式改进。

交付前

阻塞项评审

按 final-review 检查协议、卡片形态、受保护文本、事件、素材、CardSpec 对齐，失败则不能交付。

10. 对项目落地的建议

近期

补全端侧能力 manifest

围绕通勤、Todo、设备状态、健康、应用使用时长等高频卡片补充 data capability 和 event capability。

中期

端侧增加自动校验器

把 final-review 的阻塞项转成端侧校验规则，拒绝不可渲染、不可点击或路径不一致的卡片。

后续

扩展刷新和异常策略

当前 CardSpec 主要描述 dataBindings；后续可补充 refreshPlan、权限说明、失败态、lastUpdated 等运行时字段。
