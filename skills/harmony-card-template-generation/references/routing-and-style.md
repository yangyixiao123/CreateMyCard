# 模板路由与风格

## 路由输入

从 query 提取：目标尺寸、服务对象、主问题、主视觉类型、必选/可选信息、并列事实数、动作数、数据能力数、素材职责、受保护文本长度和风格意图。

## 硬过滤

按以下顺序排除模板：

1. 尺寸不一致。
2. 命中 `avoidWhen`。
3. 必选角色无法映射槽位。
4. 动作、能力、并列事实或素材超过模板上限。
5. 受保护文本超过 manifest 字符预算且无法使用声明的两行变体。
6. 所需组件不在 region 的 `allowedComponents` 中。

## 排序

对通过硬过滤的模板按以下优先级选择：

1. 主视觉语义和角色结构完全匹配。
2. 用户明确风格在 `allowedStyleProfiles` 中。
3. 无需删除可选 region。
4. `assets/templates/index.json` 中顺序更靠前。

一次只选择一个模板，不同时读取多个完整骨架进行混合。

## 风格档案

### neutral-light

浅色中性表面加一个场景强调色。适合日程、工具、设备和信息摘要。正文使用深色中性前景，强调色只用于主值、进度或动作。

### dark-focus

深色中性表面加一个高对比强调色。适合使用时长、设备状态、专注和告警。避免多个霓虹色同时竞争。

### ambient-scene

单一场景色族或受控渐变加 on-color 前景。适合天气、睡眠、倒计时和环境状态。支撑信息放入中性半透明背板。

### media-surface

主媒体或产品视觉承担识别，文字落在可验证的中性区域。适合耳机、穿戴设备和具有明确本地素材的对象；无素材时不得使用。

选择优先级为用户明确风格、query 场景、模板默认风格。用户要求的风格不在模板允许列表时，选择语义最接近的允许风格，并在最终一句说明中指出调整。最终 DSL 写 hex，不写风格档案或 token 名。

## 无匹配处理

依次缩短弱标签、删除 `shouldKeep` 槽位、尝试 `2x4`、使用同尺寸安全通用模板。仍不成立时要求简化需求，不调用其它 Skill，也不退回自由布局。
